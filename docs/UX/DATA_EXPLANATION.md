# 後台數據介面規則

1. 一般畫面只顯示使用者需要理解的名稱、狀態與結果，不直接暴露帳本、資料表或技術欄位。
2. 數據來源、抵銷關係與影響範圍必須在系統內部完整保存，僅在管理詳情、稽核或異常處理頁提供查閱。
3. 任何加減都要用自然敘述說明，例如「完成蔬食任務，獲得 10 星星」，避免使用內部術語。
4. 真實現金必須使用獨立單位、獨立帳務與獨立管理區，但不需要在無關頁面反覆提醒。
5. 玩家成長畫面應自然呈現目前等級、目前經驗、下一級門檻與尚差經驗。
6. 後台錯誤訊息要告訴操作者發生什麼事與如何處理，不能只顯示錯誤碼。
7. 高風險操作採分步確認；日常查詢、核銷與例行操作保持簡單順手。
8. 技術欄位、原始識別碼與帳本明細收在進階詳情中，首頁優先顯示營運語意。
