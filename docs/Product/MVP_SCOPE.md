# Looper MVP 範圍

## 第一階段：正式骨架

本階段目標是建立可持續擴充的產品基礎，尚不追求完整視覺與正式營運功能。

### 必須完成

- 四個 app 可獨立啟動與建置。
- 共用 packages 可被各 app 正常引用。
- 根目錄可統一執行 dev、lint、typecheck、build。
- API 提供 `/health` 健康檢查。
- 文件、資產、資料庫與 CI 目錄建立。
- GitHub Actions 驗證通過。

### 暫不包含

- 正式 LINE LIFF。
- 正式登入與權限。
- 正式資料庫與帳本。
- 真實店家核銷。
- 金流。
- 完整 Looper Forest 視覺。
- AI NPC、多人互動與大型世界。

## 第二階段：最小功能模型

建立使用者、店家、平台後台三端最小流程：

```text
使用者接任務
→ 進入待核銷
→ 店家確認
→ API 更新任務與獎勵
→ 使用者與後台讀到相同結果
```

此階段可使用 mock repository 或開發資料庫，但所有狀態更新需經由 API。

## 第三階段：有畫面的 MVP

在最小功能模型成立後，加入：

- Looper Space 主畫面。
- 土撥鼠與兔兔角色。
- 星星、能量條與任務卡。
- 任務完成回饋。
- 森林成長階段。
- 合作店家入口。
- 基本 Admin 與 Merchant 操作畫面。

## 核心驗證問題

Visual MVP 必須能回答：

1. 使用者是否能快速理解要做什麼？
2. 完成現實行動後，是否能感受到立即回饋？
3. 世界成長是否足以產生下一次回訪動機？
4. 店家核銷是否簡單、可控且不易出錯？
5. 平台是否能追蹤任務、獎勵與異常紀錄？
