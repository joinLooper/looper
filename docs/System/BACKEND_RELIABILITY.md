# 後台核心可靠性基準

## 核心原則

1. 星星、能量與核銷不可由前端自行修改。
2. 每次核銷都必須提供唯一冪等鍵，網路重送不得重複發獎勵。
3. Admin 與 Merchant API 必須分權，禁止共用高權限身份。
4. 任務接取、核銷建立與重送都必須留下審計事件。
5. 所有寫入 payload 必須經 schema 驗證並拒絕多餘欄位。
6. CI 必須執行後端測試後才能合併。

## 現階段安全邊界

目前 `x-looper-role` 僅為開發期角色邊界驗證，正式環境必須替換為經簽章驗證的 session/JWT 與伺服器端 RBAC。此 header 不得直接作為正式身份依據。

## 正式資料庫前置條件

- 核銷、獎勵帳本與審計表需使用交易寫入。
- `idempotency_key` 必須有唯一索引。
- 星星與能量異動採 append-only ledger，不直接只保存餘額。
- 所有人工補發、撤銷與修正都要保留原始紀錄與操作者。
